Getting Started With Deno

Installation

Download Deno from the official releases page and install it to a folder of your choice.

To test the installation, open a console window and run the command deno --version. If everything is ok, you should get a similar answer:

usp@home>deno --version
deno 1.17.1 (release, x86_64-pc-windows-msvc)
v8 9.7.106.15
typescript 4.5.2

Updating

By default, deno does not update anything, neither itself, nor any imported module. At first sight, this my look like a shortcoming, but actually, it ensures that existing applications are not broken accidently.

However, you can update (upgrade) deno manually by issuing the command

deno --upgrade

Version Stability and Deployment

To make sure that imported modules work with your deployed version of deno, make sure to include the version number in the module import statements:

import { Application } from "https://deno.land/x/oak@v10.5.1/mod.ts";

Hello World

Let's get our hands dirty!

  1. Create a project folder.
  2. Inside, create an empty text file named "hello-world.ts"
  3. Enter the following code: 
    console.log( "Welcome to Deno!" );
  4. Open a console window, navigate to project folder, and run the following command: 
    deno run hello-world.ts
  5. You should get the following output: 
    usp@home>deno run welcome.ts
	Welcome to Deno!
  6. Push the code to an online code repository, such as Github.
  7. Find the "raw" url to the file, and execute the following command: 
    usp@home>deno run https://raw.githubusercontent.com/BassMatiOnCode/deno/main/docs/welcome.ts
    Note that your url may vary.
  8. You should get the following output: 
    Download https://raw.githubusercontent.com/BassMatiOnCode/deno/main/docs/welcome.ts
	Check https://raw.githubusercontent.com/BassMatiOnCode/deno/main/docs/welcome.ts
	Welcome to Deno!

As you can see, the code can be loaded from anywhere, is downloaded, compiled and executed locally. The second invocation runs the compiled local program directly, without downloading.

Is that a security concern? No, it isn't, because

  1. The downloaded code runs in a V8 sandbox
  2. Without further provisions, the compiled program has no access to local resources, such as file system or network.
